# SPEC: Listado de Tareas

| Campo | Descripción y criterio de calidad |
| :--- | :--- |
| **Nombre de la feature** | Listado de Tareas de un Proyecto. |
| **Descripción general** | El usuario puede visualizar todas las tareas asociadas a un proyecto específico. Cada tarea muestra su título, estado y fecha límite. Permite filtrar las tareas rápidamente según su estado (Pendiente, En Progreso, Completada). |
| **Endpoints involucrados** | **GET /projects/{projectId}/tasks**<br> - Parámetros de ruta: `projectId`<br> - Response: array de objetos `{ id, title, estimateHours, assignee, status, finishedAt, createdAt }`<br> - Códigos: 200 OK, 404 Not Found, 500 Internal Server Error. |
| **Restricciones de negocio** | - El campo `status` solo puede tomar los valores: `TODO`, `IN_PROGRESS` o `DONE`.<br> - Si el proyecto no tiene tareas, se debe mostrar un mensaje indicando la situación en lugar de una pantalla vacía. |
| **Lineamientos técnicos** | - Angular 17+ con Standalone Components.<br> - Estilos limpios y visualización en formato de tarjetas.<br> - Comunicación con backend: encapsulada en un `TaskService`.<br> - Gestión de estado: Uso de `Signals` para almacenar el listado y manejar el filtro dinámico. |
| **Criterios de aceptación** | **Criterio 1: Carga exitosa**<br> *Dado* que el proyecto tiene tareas registradas,<br> *Cuando* el usuario navega a la ruta `/projects/:projectId/tasks`,<br> *Entonces* ve una grilla con la lista de tareas mostrando título, estado y fecha límite.<br><br> **Criterio 2: Listado vacío y Filtros**<br> *Dado* que no hay tareas para mostrar,<br> *Cuando* la base está vacía o el filtro seleccionado no arroja resultados,<br> *Entonces* el sistema muestra un mensaje claro aclarando la situación.<br><br> **Criterio 3: Filtrado por estado**<br> *Dado* que el usuario está viendo el listado de tareas,<br> *Cuando* selecciona el filtro "Pendiente" (TODO),<br> *Entonces* la vista se actualiza instantáneamente para mostrar únicamente esas tareas.<br><br> **Criterio 4: Manejo de Errores**<br> *Dado* que ocurre un problema de conexión con el backend,<br> *Cuando* el usuario intenta acceder a la vista,<br> *Entonces* el sistema oculta la grilla y muestra una tarjeta de error indicando el fallo. |
